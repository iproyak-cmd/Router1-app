# Fabula Daily Look Editorial Standard

## Неприкосновенное правило

Каждый `look_id` и каждое изображение используются для конкретной установки
только один раз. История показов не обнуляется. Старый образ нельзя вернуть в
ротацию под новым идентификатором.

## Редакционный ритм

- В опубликованном каталоге всегда должно быть не меньше восьми новых образов.
- Пользователю предлагаются только восемь образов из `active_batch_id`; новый
  пользователь не начинает со старого архива.
- На шестой день партии создаётся задача на следующую партию из восьми образов.
- Новая партия публикуется до исчерпания двух оставшихся резервных дней.
- Добавление партии не требует выпуска новой версии приложения: приложение
  получает актуальный манифест с `router1.tech`.

## Арт-дирекшен

- Уровень: luxury fashion editorial, а не каталог одежды и не сток.
- Каждый образ отличается героиней, сценой, светом, палитрой и стилизацией.
- Визуал должен побуждать расти, собираться, развивать вкус и брать пример.
- Только оригинальные изображения без брендов, текста, водяных знаков,
  инфлюенсерских клише, дешёвого гламура и повторного использования.
- Композиция оставляет спокойное поле слева под редакционный текст Fabula.

## Публикация партии

1. Создать восемь новых файлов в `assets/fabula/daily_looks/`.
2. Присвоить каждому новый неизменяемый `look_id`.
3. Добавить записи в `site/fabula/content/daily-looks.json`, не удаляя и не
   изменяя историю: workflow проверяет каталог как append-only.
4. Назначить новую партию в `active_batch_id`.
5. Перенести `next_batch_due` на шестой день новой партии.
6. Пройти автоматическую проверку уникальности и редакционного резерва.
