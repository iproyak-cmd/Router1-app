## 1. Что такое Router1 (бизнес)

- VPN-сервис на базе **WireGuard + Xray/VLESS Reality + AmneziaWG**. Продаём **результат**
  («интернет, который просто работает»), а не технологию.

  (видео/мессенджеры/нейросети). Гаджеты/ноутбуки/ПК — вторичный продукт.
- **Продукты:** роутер (главный), смартфон, ноутбук/ПК, тест на сутки за 20₽.
- **⚠️ ЗАПРЕЩЕНО** на сайте, в боте и приложении: слова «VPN», «YouTube», «обход блокировок».
  Пишем нейтрально: «интернет, который работает», «видео и стриминги», «нейросети», «мессенджеры».
- **Цель Виктора (владелец):** стабильные $7–10k/мес. Он предприниматель, НЕ программист.
  Ждёт работу уровня Senior DevOps + Senior Python: автономно, с кратким отчётом после изменений

## 6. Git и сборка APK

**Git:** `router1_app_mvp` под git (baseline 3c8645d). Конвенция коммитов: префикс `codex:` / `claude:`.
`.gitignore` исключает build//.dart_tool//*.apk/*.backup*/.env. Перед сборкой — коммит.
Инкремент версии синхронно в `pubspec.yaml` (`version:`) И `main.dart` (`router1AppVersion`).
⚠️ У Codex может не быть своей git-identity — настрой `git config user.email codex@router1.local`,
не коммить под чужим именем.

**⚠️ CRLF:** если правишь Dart-файлы с Windows-тулинга — на сервере получаются CRLF, git видит
весь файл изменённым. После заливки на сервер: `sed -i 's/\r$//' lib/main.dart` (вернуть LF).

**Сборка APK (точная команда — КРИТИЧНО):**
```
systemd-run --unit=r1buildN --collect \
  --property=WorkingDirectory=/root/router1_app_mvp \
  --setenv=CI=true --setenv=HOME=/root --setenv=GRADLE_OPTS='-Dorg.gradle.daemon=false' \
  /root/flutter/bin/flutter build apk --release --target-platform android-arm64 \
  --dart-define=ROUTER1_APP_TOKEN=<токен из /opt/vpn_bot/.env APP_API_TOKEN>
```
- **`--dart-define=ROUTER1_APP_TOKEN=...` ОБЯЗАТЕЛЬНО.** Без него приложение получает 401 на все
  API → показывает оплату даже оплатившим.
- **`--target-platform android-arm64` ОБЯЗАТЕЛЬНО.** Без него APK 57МБ (3 архитектуры) > лимита
  Telegram 50МБ. С ним — 19МБ.
- Не запускать два build параллельно (Gradle-локи). NL ~3.8ГБ RAM — при активном Codex Gradle
  может словить OOM (в `android/gradle.properties` стоит `jvmargs=-Xmx1400m`, `workers.max=1`).

**Деплой релиза (каждый раз ВСЕ 3 шага):**
1. APK → `/var/www/html/app/router1.apk` на MSK (бэкап старого в `app/backup/`).
2. Обновить `/var/www/html/app/version.json` (version/build/url/notes) — приложение сравнивает build
   и показывает баннер обновления.
3. Отправить APK Виктору в `@Router1_Claude_bot` (`sendDocument`, chat_id 6800777154, с NL).

## 7. Критические правила (нарушение = инцидент)

1. **BACKUP перед любым изменением** бота/WG/Xray/nginx/SQLite/systemd. Бэкапы — в
   `/opt/vpn_bot/backups/<метка>/`. DELETE/DROP/TRUNCATE — только с бэкапом БД.
2. **Диагностика → причина → фикс → проверка.** Никогда наоборот.
3. **НИКОГДА не тестируй сетевые изменения на живом клиенте Виктора** (был инцидент 08.07:
   tproxy на боевом пире → у клиента пропал весь интернет). Только на отдельном тест-пире.
4. **`systemctl start` → сразу `systemctl enable`** (см. инцидент FR/xray 10.07).
5. После правки Python в `/opt/vpn_bot/` — `venv/bin/python -m py_compile <файл>`, потом
   `systemctl restart` нужных сервисов (vpn-bot / router1-site-api / platega-webhook), проверить
   `is-active` + `/api/health`.
6. Сторонние Flutter-пакеты для файлов/железа проверяй по НАТИВНОМУ коду (Kotlin/Swift), а не по
   документации (был баг file_saver: `saveFile()` писал в приватную папку вопреки докам).

  `awg show`/`wg show`, после теста — снять пир и удалить строки БД.

- **Приложение:** `flutter analyze lib/main.dart` перед сборкой; APK-флоу до экрана оплаты можно
