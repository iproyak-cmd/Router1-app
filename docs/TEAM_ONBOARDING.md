## Что такое Router1

VPN-сервис на базе WireGuard/AmneziaWG + Xray, но **продаётся не как VPN**, а как
«умный интернет для дома» — приложение настраивает роутер Keenetic за 5 минут и держит
нужные сервисы (соцсети, видео, нейросети, игры) быстрыми и доступными.
Юридическая осторожность: НИГДЕ в клиентских текстах (сайт/бот/приложение) не упоминаем
слова VPN, обход блокировок, названия конкретных заблокированных сервисов открытым текстом.

Продукт из трёх частей, в порядке приоритета:
1. **Мобильное приложение (Flutter, Android)** — главный продукт. Само находит роутер,
   настраивает, следит за соединением.
2. **Telegram-бот** (@router1_lk_bot) — личный кабинет, оплата, поддержка, часть воронки.
3. **Сайт** (router1.tech) — витрина, тоже часть воронки, ведёт на скачивание приложения.

Реальные деньги, реальные клиенты. Стабильность важнее новых фич.

## Приложение — единственный git-репозиторий проекта

Обязательно прочитай `docs/BUILD_CONVENTION.md` в этом репозитории — правила коммитов,
нумерации версий и именования APK.

Кратко:

- Коммит после правки: `git add -A && git commit -m "<claude|codex|gemini>: <что>"`.
- Версия: `pubspec.yaml` `version: X.Y.Z+BUILD` и `router1AppVersion` в `lib/main.dart`

### Сборка APK
```
systemd-run --unit=r1bN --collect \
  --property=WorkingDirectory=/root/router1_app_mvp \
  --setenv=CI=true --setenv=HOME=/root \
  --setenv=GRADLE_OPTS='-Dorg.gradle.daemon=false' \
  /root/flutter/bin/flutter build apk --release \
  --target-platform android-arm64 \
  --dart-define=ROUTER1_APP_TOKEN=<токен из /opt/vpn_bot/.env APP_API_TOKEN>
```
Важно:
- `--target-platform android-arm64` ОБЯЗАТЕЛЕН — без него APK получается ~57МБ
  (3 архитектуры разом) и не проходит лимит Telegram sendDocument (50МБ). С флагом — ~19МБ.
- NL — слабый сервер (3.8ГБ RAM). Если Codex тоже активен, сборка может поймать OOM.
  В `android/gradle.properties` уже стоят лимиты (`-Xmx1400m`, `workers.max=1`) — не убирай.
- Не запускать два `flutter build` параллельно — ловит Gradle-локи.

### Доставка APK Виктору
Только с NL (российские сервера не достают `api.telegram.org`). Через бота
`@Router1_Claude_bot` (токен и chat_id — см. историю памяти/чата), `curl -F document=@...
sendDocument`. Публикуем И на сайт: `/var/www/html/app/router1.apk` на MSK — это то, что
реально качают клиенты через сайт/бот, обновляй его при каждом релизе, иначе клиенты
скачивают старьё (уже был такой баг 09.07 — файл висел с 7 июля).

### Автообновление приложения
Механизм: `https://router1.tech/app/version.json` — статический файл
`{"version":"X.Y.Z","build":N,"url":"...","notes":"..."}`. Приложение при открытии
дашборда сверяет свою версию, показывает баннер если на сервере новее. Обновляй этот
файл при каждом релизе (build должен расти).

## Split-tunnel — как устроена маршрутизация роутера

Приложение настраивает Keenetic на 3 режима (см. `Router1RouteProfileKind` в
`router1_api.dart`): **Standard** (YouTube/Telegram/WhatsApp через тоннель), **+AI**
(плюс нейросети), **For Gamers** (плюс игровые сервисы). Технически различаются только
route-profile (какие домены/IP роутер шлёт в тоннель), не туннелем как таковым.

Нейросети (ChatGPT особенно) — сложный случай: упираются в Cloudflare-антибот
(`cf-mitigated: challenge`), не в бан IP. Решение — Xray-переоткрытие с TLS-отпечатком
Chrome на выходном узле (работает на NL, не доведено до конца на FR — см. память/чат
за 07-08.07 если понадобится продолжить).

## Реферальная/партнёрская система (готова, Codex построил, я подключил к приложению 09.07)

Два независимых механизма, не путать:
1. **Блогеры** (`referral_partners` таблица) — 15% комиссия постоянно с каждого клиента.
   Создаются командой в боте «➕ Создать ссылку блогеру». Ссылка вида
   `router1.tech/?r=КОД` или `t.me/router1_lk_bot?start=КОД`. Статистика: `/api/partner/stats`.
2. **Клиент приглашает друга** (`referral_events` таблица) — разовый бонус +15 дней
   рефереру, когда приглашённый первый раз оплатит. Формат ссылки:
   `t.me/router1_lk_bot?start=REF_КОД`, где КОД = `R1` + id клиента, дополненный нулями
   до 6 цифр (см. `referral_code_for()` в `router1/db.py`).

Сайт (`?r=` в URL) и приложение (поле «Промокод» на оплате) теперь передают код дальше
на `/api/order` полем `ref_code` — бэкенд уже это принимал, просто раньше никто не отправлял.

## Watchdog (мониторинг, мой, на 2.59.41.90)

`/opt/router1_watchdog/watchdog.py`, systemd-таймер каждые 2 минуты. Проверяет все
сервера/сервисы, автоматически перезапускает упавшие (кроме самих VPN-интерфейсов —
только алерт, ручная починка). Алерты — в Telegram Виктору через relay (прямой доступ
к Telegram API с RU-серверов не работает).
